NAME
     om_project - project a distribution model using the openModeller framework

SYNOPSIS
       om_project [-] v --version | r --xml-req XML_REQUEST_FILE m --dist-map FILE [ --log-level LEVEL ] [ --log-file FILE ] [ --prog-file FILE ] [ --stat-file FILE ]

DESCRIPTION
       om_project is a command line tool to project distribution models. The main input is an XML file containing a projection request according to the ProjectionParameters element definition in http://openmodeller.cria.org.br/xml/1.0/openModeller.xsd (see also projection_request.xml in the openModeller examples directory). The distribution map (raster file) generated by openModeller will be stored in another file specified as a parameter. To generate a distribution model, which is part of the projection parameters, use om_model.

OPTIONS
       -v, --version     Display version info.

       -r, --xml-req     File containing a projection request in XML.

       -m, --dist-map    File where the generated raster will be stored.

       --log-level       openModeller log level: debug, warn, info or error. Defaults to "info".

       --log-file        File to store log information.

       --prog-file       File to store progress.

       --stat-file       File to store projection statistics (in XML).

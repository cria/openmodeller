openModeller Introduction

Note: This document is not intended to be a replacement for the manual pages nor for the documentation available in openModeller website. Instead, this is a quick introduction on how to generate a model using the CLI (command line interface) tools available on every openModeller distribution.

You have just installed openModeller. Here you'll find some instructions to get you into the basics on how to generate a simple model using the command-line interface tools available.

openModeller Modularity

openModeller itself isn't a standalone software. It's a modular software, in which many parts work together to generate its functionality. A simplified model of openModeller structure is shown below:


There are different ways to use its core modelling functionality. If you are interested in a graphical user interface (GUI) with buttons, controls and images you should download a different package called openModeller Desktop. Instead, if you want to use the CLI interface, openModeller offers some specific tools (commands) to use all the funcionality the GUI offers (and sometimes, even more). The point here is that you can effectively use openModeller in a variety of ways.

Now, let's get our hands dirt.

Generating a model

Note: This document focuses on the Windows platform to show how to use a terminal, however, this should just apply for any other platforms as well.

In Windows, the terminal is the Command prompt, the MS-DOS prompt. Starting from openModeller 1.3.0, installer includes shortcuts to all the commands available. When you click in one of these shortcuts, a new terminal is opened called openModeller Terminal, which is basically a standard Command prompt, with a usage help that shows the options to effectively use the command.

The openModeller Terminal starts in the Working Folder specified at the installation. In its inside, there is a folder called examples, which we'll use to run our... example.

From now on, we make some assumptions here. The commands you should type will be in bold, while the rest of the monospaced font is the output from the terminal itself and/or openModeller. Also, we assume the Working Folder to be C:\om.

So, let's enter the examples directory:

C:\om> cd examples
C:\om\examples>

Now we will issue the command to generate a model itself. Here, as a matter of the example, we'll use a simple algorithm (faster) and also the default parameters of openModeller for the chosen algorithm.

The request.txt file contains many definitions that om_console use to generate its model. For instance, Map specifies the layers it uses, Output file is the file generated, and so on. You can modify this file to fit your own needs, or you can generate a completely different one.
